TEMPLATE STRUCTURE
-------------------
All templates follow this structure:

1. Extend base.html:
   {% extends 'base.html' %}

2. Define content block:
   {% block content %}
   ... template content ...
   {% endblock %}

3. Optional blocks:
   {% block title %}Page Title{% endblock %}
   {% block extra_css %}{% endblock %}
   {% block extra_js %}{% endblock %}

TEMPLATE VARIABLES
------------------
Common variables available in templates:
- user: Current authenticated user
- messages: Django messages framework
- form: Form object (in form templates)
- request: HTTP request object

OTP TEMPLATE SPECIFIC VARIABLES
-------------------------------
In verify_otp.html:
- username: Username for verification
- messages: Success/error messages
- form: OTP input form (if using form)

TEMPLATE TAGS USED
------------------
- {% extends %}: Template inheritance
- {% block %}: Define block content
- {% url %}: Generate URLs
- {% csrf_token %}: CSRF protection
- {% if %}: Conditional statements
- {% for %}: Loop statements
- {{ variable }}: Display variables

STATIC FILES IN TEMPLATES
--------------------------
- Bootstrap CSS: CDN link in base.html

- Bootstrap Icons: CDN link in base.html
- Custom CSS: Inline styles in base.html

TEMPLATE CUSTOMIZATION
----------------------
To customize templates:

1. Base Template:
   - Edit: templates/base.html
   - Changes affect all pages

2. App Templates:
   - Edit: app_name/templates/app_name/template_name.html
   - Changes affect specific pages

3. Add Custom CSS:
   - Use {% block extra_css %} in child templates
   - Or add to base.html <style> tag

4. Add Custom JavaScript:
   - Use {% block extra_js %} in child templates
   - Or add to base.html before </body>

TEMPLATE BEST PRACTICES
-----------------------
1. Always extend base.html
2. Use Bootstrap classes for styling
3. Include CSRF token in forms
4. Display messages to users
5. Validate forms on frontend and backend
6. Use semantic HTML
7. Make templates responsive
8. Test on different screen sizes

TROUBLESHOOTING TEMPLATES
-------------------------
Issue: Template not found
Solution: Check TEMPLATES['DIRS'] in settings.py

Issue: Base template not found
Solution: Ensure base.html is in templates/ directory

Issue: Template variables not displaying
Solution: Check view context and template variable names

Issue: Static files not loading
Solution: Run: python manage.py collectstatic

Issue: Forms not working
Solution: Ensure {% csrf_token %} is included in form
